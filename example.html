Quick guide for the cwColorButton function of colorWheel

cwColorButton("[NAME]",[R],[G],[B],[A],"[Syle]","[Side X]","[Side Y]","[Alignment]","[Visibility]","[Rounding]");

[NAME]

Button name

[R]

0 - 255

[G]

0 - 255

[B]

0 - 255

[A]

1 = 0%
0,5 = 50%
0 = 100%

[Style]

There are 3 possibilities for the main appearance of the button:
1.) onlyButton
2.) colorButton
3.) values

[Side X]

X-Side with e.g. the ending in PX = 100px

[Side Y]

Y-Side with e.g. the ending in PX = 10px

[Alignment]

The alignment of the button text has 3 possibilities:
1.) left
2.) right
3.) centre

[Visibility]

If the value is left blank here, the color data field always appears, this is intended for the development of new pages for control; in the final version one should then select the option "hidden". With the option "hidden", the color values are still entered in this field (important for a query or the transmission via form), but this remains invisible for the user.

[Rounding]

Here you can additionally specify the edge rounding of the buttons, for a variation difference with identical use of given CSS.



Important

Of course, it is important that the respective CSS as well as Javascripts do not collide with the functions of cwColorButton, therefore you should take a look at the code before using it to avoid later stumbling blocks!
